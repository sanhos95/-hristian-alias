# Alias (біблійні слова) — Vue 3 + Vite + Tailwind

Мінігра у стилі **Alias**: пояснюєш слово/ім’я, команда (або один гравець) відмічає **“Вгадано”** або **“Пас”**. Проєкт зібраний як простий SPA на **Vue 3**.

## Швидкий старт

### Вимоги
- **Node.js** (рекомендовано LTS)
- **npm**

### Запуск

```bash
npm install
npm run dev
```

### Збірка

```bash
npm run build
npm run preview
```

## Як грати (правила)

- **Тривалість раунду**: за замовчуванням 1:30 (90 секунд).
- **Початок**: перше слово не показується, доки не натиснути **“Старт”**.
- **Оцінювання**:
  - **Вгадано** → +1
  - **Пас** → +1 “пас” (потім у рахунку віднімається)
  - **Рахунок** = `вгадано - пас`
- **Фініш при 0:00**:
  - коли таймер доходить до 0, гра **не скидає** слово автоматично
  - дозволяє **дозавершити поточне слово**
  - після натискання **“Вгадано”** або **“Пас”** на останньому слові — хід завершується і відкривається модалка результату

## Екрани (2 “сторінки”)

Додаток має два основні стани (`screen`):

- **`setup` (Налаштування)**: вибір категорій, режиму, команд, кількості гравців, кнопка “Почати гру”.
- **`game` (Гра)**: таймер, картка слова з бейджем типу, кнопки результату, історія, модалка фіналу, кнопка повернення в налаштування.

> Навігація — умовний рендер в одному компоненті (`App.vue`), без роутера.

## Режими

### Одиночний (`mode = 'solo'`)
- Просто граєш раунди; кожен старт очищає локальний стан раунду.

### Командний (`mode = 'team'`)
- Є **дві команди** (назви вводяться).
- Для кожної команди задається **кількість гравців** (`playersA`, `playersB`) — може бути різна.
- Після завершення ходу в модалці доступна кнопка **“Передати хід іншій команді”**:
  - перемикається активна команда
  - індекс гравця в активній команді збільшується по колу
  - стартує новий хід з чистими лічильниками (вгадано/пас/історія) і новим таймером
- У модалці показується **поточний результат ходу** та **загальний рахунок команд**.

## Категорії та “тип” картки

### Категорії
Категорія визначає пул слів, з якого береться наступна картка:
- `names` — біблійні імена
- `cities` — біблійні міста
- `events` — біблійні події
- `stories` — біблійні історії
- `all` — змішаний режим (об’єднаний пул з усіх категорій)

### Бейдж типу (для змішаного режиму)
Над словом показується бейдж **“Імʼя / Місто / Подія / Історія”**. Він визначається **по значенню слова** (через `Set` для кожного словника), тому працює навіть коли обрана `all`.

## Джерела даних (словники)

Словники лежать у `src/data/`:
- `src/data/bibleNames.js`
- `src/data/bibleCities.js`
- `src/data/bibleEvents.js`
- `src/data/bibleStories.js`

Формат простий: `export const ... = ['...', '...']`.

### Як додати слова
1. Відкрий потрібний файл у `src/data/`.
2. Додай рядки в масив.
3. Перезапусти dev-server, якщо потрібно (зазвичай HMR підхоплює автоматично).

### Як додати нову категорію
1. Створи файл у `src/data/`, наприклад `bibleBooks.js`.
2. Імпортуй його в `src/App.vue`.
3. Додай категорію в `pool` (computed) та в UI налаштувань.
4. Додай `Set` для бейджа (щоб працювало у `all`) і розшир `cardBadge`.

## Вибір наступної картки (anti-repeat)

Логіка вибору “наступного слова”:
- є **локальний набір** `seen` — для уникання повторів в межах категорії
- є **глобальний набір** `globalSeen` — щоб на цьому пристрої не повторювати слова між раундами
- якщо слова вичерпались (немає `remaining`) — виставляється `exhausted` і гра завершується

> Це зроблено максимально просто: без серверної синхронізації й без “перемішування мішка” наперед.

## Модалка результатів

Модальне вікно відкривається при завершенні ходу:
- по кліку на фон (оверлей) — закривається
- по “×” — закривається
- по **Esc** — закривається

В модалці:
- рахунок ходу
- вгадано / пас
- (у командному режимі) хто грав + загальний рахунок команд + кнопка “Передати хід”

## Збереження стану (localStorage)

Проєкт зберігає ключовий стан у `localStorage`, щоб при перезавантаженні сторінки:
- не втрачати налаштування
- (за потреби) відновлювати прогрес

Збереження відбувається через `watch(...)` у `App.vue`.

## Структура проєкту

```
src/
  App.vue              # вся логіка гри + UI
  main.js              # bootstrap Vue
  style.css            # tailwind import
  data/                # словники
```

## Технології
- **Vue 3** (Composition API, `<script setup>`)
- **Vite**
- **Tailwind CSS v4** через `@tailwindcss/vite`

## Розробницькі примітки
- Проєкт спеціально зроблений **в одному компоненті** (`App.vue`) — щоб було легше переносити/рефакторити.
- Якщо планується розвиток, рекомендовано винести:
  - словники/логіку гри в `composables/`
  - UI блоки (модалка, таймер, історія) у `components/`
